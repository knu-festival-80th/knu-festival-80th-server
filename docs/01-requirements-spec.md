# 사용자 요구사항 명세서 (SRS)

> **프로젝트**: 2026 경북대학교 80주년 대동제 웹앱 서비스 (백엔드)  
> **버전**: v1.1  
> **최종 수정일**: 2026-04-25  
> **목적**: Validation 기준 문서 — "사용자가 원하는 것을 올바르게 정의했는가?"

---

## 1. 문서 개요

이 문서는 축제 이용자(학생, 외부 방문객)와 운영자(주막 관리자, 축제 운영 주체)의 관점에서 시스템이 **무엇을 해야 하는지**를 정의한다.  
기술적 구현 방법은 다루지 않으며, 사용자가 기대하는 행위(behavior)를 중심으로 기술한다.

## 2. 이해관계자(Stakeholder)

| 구분 | 역할 | 주요 관심사 |
|------|------|-------------|
| 일반 사용자 | 축제 방문객 (학생, 외부인) | 공연/부스 정보 확인, 참여형 콘텐츠 |
| 주막 방문 손님 | 주막 이용을 원하는 손님 | 웹에서 대기 등록, 호출 알림, 남은 대기 확인, 본인 취소 |
| 주막 운영자 | 대동제 주막 담당자 | 메뉴 관리, 대기열 관리, 품절 처리 |
| 축제 운영 주체 | 대행사 또는 총학생회 | 긴급 공지, 축제 일정 관리, 전체 모니터링 |
| 개발팀 | 백엔드/프론트엔드 개발자 | 명확한 요구사항, API 명세 |

## 3. 사용자 요구사항

### 3.1 메인 페이지 (HOM)

#### REQ-HOM-01: 축제 정보 및 카운트다운

| 항목 | 내용 |
|------|------|
| **설명** | 축제 시작/종료 일시, 현재 상태를 제공하고 카운트다운을 지원한다. |
| **사용자 스토리** | 방문객으로서, 축제까지 남은 시간과 현재 진행 상태를 한눈에 확인하고 싶다. |
| **인수 조건** | 1. 축제 시작/종료 일시를 서버에서 내려줌 (프론트 하드코딩 방지) <br> 2. 축제 상태를 제공 (시작 전 / 진행 중 / 종료) <br> 3. 관리자가 축제 일정을 수정할 수 있음 |
| **우선순위** | P1 (필수) |

#### REQ-HOM-02: 실시간 부스 랭킹

| 항목 | 내용 |
|------|------|
| **설명** | 주막/부스의 '좋아요' 수를 집계하여 순위를 실시간 반영한다. |
| **사용자 스토리** | 방문객으로서, 인기 있는 주막을 빠르게 찾고 싶다. |
| **인수 조건** | 1. 좋아요 수 기반 실시간 순위 표시 <br> 2. 정렬 옵션 지원 (좋아요순 / 대기 적은 순) <br> 3. 부스당 좋아요는 1인 1회 제한 |
| **우선순위** | P1 (필수) |

#### REQ-HOM-03: 공연 정보 제공

| 항목 | 내용 |
|------|------|
| **설명** | 축제 기간 내 공연의 일시, 장소, 출연진 정보를 제공한다. |
| **사용자 스토리** | 방문객으로서, 현재 진행 중인 공연과 다음 공연 일정을 알고 싶다. |
| **인수 조건** | 1. 공연 일시, 장소(무대명 + 위치), 출연진 정보 표시 <br> 2. 현재 시각 기준 진행 중/직후 공연 강조 표시 <br> 3. 날짜별 필터링 지원 |
| **우선순위** | P1 (필수) |

#### REQ-HOM-04: 공지사항

| 항목 | 내용 |
|------|------|
| **설명** | 운영 주체가 발행하는 공지사항을 제공하며, 긴급 공지는 상단에 고정 노출한다. |
| **사용자 스토리** | 방문객으로서, 긴급 상황(장소 변경, 취소 등)이나 일반 안내를 즉시 알고 싶다. |
| **인수 조건** | 1. 긴급 공지는 메인 페이지 상단 고정 <br> 2. 공지 유형 구분 (긴급 / 일반 / 날씨 / 셔틀 등) <br> 3. 유효 기간이 지난 공지는 자동 비노출 |
| **우선순위** | P2 (중요) |

### 3.2 이용자 기능 (USR)

#### REQ-USR-01: 캠퍼스 지도 서비스

| 항목 | 내용 |
|------|------|
| **설명** | GPS 기반 현재 위치 표시 및 주막/공연장 위치를 지도에 표시한다. |
| **사용자 스토리** | 외부 방문객으로서, 캠퍼스 내 주막과 공연장 위치를 지도에서 확인하고 싶다. |
| **인수 조건** | 1. 사용자 현재 위치 GPS 마커 표시 <br> 2. 주막/공연장 아이콘 및 위치 좌표 표시 <br> 3. 아이콘 클릭 시 상세 정보 모달 (이름, 메뉴, 대기 현황) <br> 4. 리뷰 작성 기능 (별점 + 텍스트) |
| **우선순위** | P1 (필수) |

#### REQ-USR-02: 주막 온라인 대기열 (웹 등록)

| 항목 | 내용 |
|------|------|
| **설명** | 손님이 축제 웹앱에서 부스 정보를 조회한 뒤 직접 대기 등록하는 온라인 웨이팅 시스템이다. 별도의 현장 태블릿은 두지 않으며, 모든 손님 행위(등록/현황 확인/내 대기 조회/취소)는 웹에서 이뤄진다. |
| **사용자 스토리 (손님)** | 축제 방문객으로서, 웹에서 주막 정보를 보고 그 자리에서 대기 등록을 한 뒤 차례가 오면 문자로 알림을 받고, 사정이 생기면 직접 대기를 취소하고 싶다. |
| **사용자 스토리 (운영자)** | 주막 운영자로서, 대기 손님 목록을 실시간으로 보고 호출/입장/취소를 관리하고 싶다. |
| **인수 조건 (손님 화면 — 웹)** | 1. 웹에서 이름·인원수·전화번호 입력으로 대기 등록 <br> 2. 등록 완료 시 대기 번호 발급 + 현재 앞 대기팀 수 + 예상 대기 시간 표시 <br> 3. 부스별 현재 대기 현황(남은 팀 수)을 누구나 실시간 조회 가능 <br> 4. 본인 대기 상태 조회(전화번호 뒤 4자리 본인 확인) <br> 5. 본인이 직접 대기 취소 가능(전화번호 뒤 4자리 본인 확인) <br> 6. 동일 전화번호로 동일 부스에 중복 등록 불가 |
| **인수 조건 (운영자 — 관리자 화면)** | 1. 전체 대기팀 목록 실시간 확인 (상태별 필터) <br> 2. 호출 시 SMS 발송, 5분 내 미방문 시 자동 건너뛰기(SKIPPED) <br> 3. 대기열 중간에 손님 추가 (순서 지정 삽입) <br> 4. 대기열 중간에서 특정 손님 제거 또는 순서 변경 <br> 5. 대기 접수 ON/OFF (영업 종료 시 신규 등록 차단) |
| **우선순위** | P1 (필수) |
| **담당** | 상민 (SMS API 연동) |

#### REQ-USR-03: 80주년 실시간 롤링페이퍼

| 항목 | 내용 |
|------|------|
| **설명** | 다수의 사용자가 하나의 가상 도화지에 실시간으로 방명록/그림을 남긴다. |
| **사용자 스토리** | 방문객으로서, 축제 기념으로 거대한 디지털 캔버스에 메시지를 남기고 싶다. |
| **인수 조건** | 1. 다수 사용자 동시 접속 및 실시간 드로잉/텍스트 입력 <br> 2. 스티커 배치 (사용자당 개수 제한) <br> 3. 모바일 브라우저에서 원활한 렌더링 <br> 4. 화면에 보이는 영역의 데이터만 로딩 (Viewport 기반) |
| **우선순위** | P2 (중요) |
| **담당** | 우석 |

#### REQ-USR-04: 두근두근 인스타팅 (랜덤 매칭)

| 항목 | 내용 |
|------|------|
| **설명** | 성별과 Instagram ID를 입력하면 이성의 ID를 1:1 랜덤 매칭한다. |
| **사용자 스토리** | 축제 참여자로서, 재미있는 랜덤 매칭으로 새로운 사람을 만나고 싶다. |
| **인수 조건** | 1. 성별 + Instagram ID + 비밀번호 입력으로 참여 <br> 2. 축제 기간 중 1회 참여 제한 (중복 불가) <br> 3. 지정 시간(예: 22시)에 일괄 결과 오픈 (Time Drop) <br> 4. 미매칭 잔여 인원은 즉석 공개 방식 전환 <br> 5. 스크래치 UI로 결과 확인 (서버가 공개 여부 제어) <br> 6. Instagram 딥링크로 프로필 바로가기 <br> 7. 성별 불균형 시 매칭 일시중단 + 안내 메시지 <br> 8. 영문 UI 지원 (외국인 유학생) <br> 9. 매칭 신청 취소 가능 |
| **우선순위** | P2 (중요) |
| **담당** | 동현 |

#### REQ-USR-05: 웹 포토부스 & 호반우스타그램

| 항목 | 내용 |
|------|------|
| **설명** | 웹 기반 포토부스에서 사진 촬영/편집 후 실시간 피드에 공유한다. |
| **사용자 스토리** | 방문객으로서, 축제 프레임이 적용된 사진을 찍고 다른 참여자들과 공유하고 싶다. |
| **인수 조건** | 1. 카메라 연동 또는 갤러리 업로드 <br> 2. 호반우 전용 프레임/스티커 적용 (1장 또는 4장 모드) <br> 3. 촬영 시 프레임이 화면에 미리 표시 <br> 4. 결과물 다운로드 및 Instagram 스토리 공유 <br> 5. 플랫폼 내 공유 피드 업로드 <br> 6. 피드 게시물 좋아요 + 좋아요 취소 기능 <br> 7. 좋아요 1위 리워드 (주기: 1일 또는 3일, 관리자 설정) <br> 8. 업로드 시 Instagram ID 선택 기입 + 딥링크 |
| **우선순위** | P2 (중요) |

## 5. 용어 정의

| 용어 | 정의 |
|------|------|
| 주막 | 대동제에서 운영되는 음식/음료 판매 부스 |
| 호반우 | 경북대학교 마스코트 |
| Time Drop | 사전 신청을 축적 후 지정 시간에 일괄 매칭 결과를 공개하는 방식 |
| 호반우스타그램 | 축제 전용 사진 공유 피드 (Instagram 유사 서비스) |
| 인스타팅 | Instagram + 소개팅의 합성어, 랜덤 매칭 서비스 명칭 |
| 온라인 웨이팅 | 축제 웹앱에서 손님이 직접 대기 등록·조회·취소까지 수행하는 대기열 모델 |
| SKIPPED | 호출 후 제한 시간(5분) 내 미방문 시 자동으로 건너뛰어지는 상태 |
